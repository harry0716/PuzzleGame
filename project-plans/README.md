# Project Plans

This folder stores proposal-oriented documents that grow alongside the main project.

Its purpose is different from the core technical documentation:

- the root project docs explain how the system works
- this folder explains how the system can be positioned, proposed, extended, and
  developed into future science-popularization, teaching-practice, and research work

## Current Planning Tracks

- `plan-v1-science-popularization.md`
  AI cross-disciplinary science-popularization positioning draft
- `plan-v1-teaching-practice.md`
  teaching-practice and curriculum-integration positioning draft
- `plan-v1-research-concept.md`
  early-stage research and project-concept positioning draft

## Recommended Usage

- keep proposal versions instead of overwriting one file
- when the platform gains a major new scene, module system, difficulty layer, or
  activity-management feature, update the relevant planning files here
- use this folder as the basis for future grant proposals, competition submissions,
  outreach planning, slide decks, or paper-writing materials

## Relationship To The Main Project

This folder is a planning track, not a replacement for:

- `codex.md`
- `README.md`
- `DELIVERY.md`
- `VERSION_RECORD.md`

Those files remain the main technical and implementation record.

## Current Maturity Snapshot

At the current stage, this repository already has enough depth to support:

- a science-popularization project path
- a teaching-practice project path
- an early research / proposal-development path

The game system is no longer only a single quiz scene. It now includes:

- multiple live scenes
- a module-based fifth scene
- activity routing with scene / module / difficulty / locked mode
- presenter-side event control
- authored independent difficulty scripts on initial pilot targets

This folder should therefore be maintained as a long-term writing asset in
parallel with the technical product.
