# Third Scene Draft v0.2

## 場景定位

- Scene ID: `smart-factory-mission-station`
- 場景名稱：`智慧工廠任務站`
- 風格：任務闖關型，搭配職涯角色結果卡
- 玩家身份：智慧工廠任務員
- 核心訊息：
  電子工程不只存在於晶片或展示設備中，也存在於感測器、控制器、
  檢測系統與自動化產線的協同運作裡。

## 場景故事

玩家進入一個智慧工廠任務站。現場有一條小型示範產線出現異常訊號，
玩家必須協助判斷：

- 哪個警示最值得先處理
- 哪個設備在執行什麼角色
- 當品質或流程出現異常時，正確的反應順序是什麼
- 為什麼這些工作都屬於電子工程師的舞台

這個場景不應該像課堂考卷，而應該像一個「幫工廠恢復穩定運作」的小型任務。

## 與前兩個場景的區隔

- `晶片獵人`
  偏 AI、晶片、設備與系統認識
- `電子工程雙體驗`
  偏展攤沉浸體驗、飛控、VR、訊號整合
- `智慧工廠任務站`
  偏自動化、感測、控制、檢測、產線邏輯

第三場景的重點是：

- 更工業現場
- 更像系統維持與流程判斷
- 更能凸顯電子工程在產線中的穩定性與整合能力

## 題型配置

維持 7 個互動節點：

1. `single-choice`
2. `branching`
3. `timed-choice`
4. `image-choice`
5. `matching`
6. `ordering`
7. `single-choice`

## 流程總覽

1. 開場題：理解什麼是智慧工廠
2. 分支題：先查感測器警示，或先查產線流程
3. 限時題：快速判斷最需要優先處理的異常
4. 圖片題：辨識最關鍵的工廠設備 / 檢測站 / 感測區
5. 配對題：把設備和功能配對
6. 排序題：排出正確的異常處理流程
7. 收束題：回到電子工程在智慧工廠中的核心角色

## 題目草案

### Q1

- Type: `single-choice`
- 題目主軸：什麼讓工廠變得「智慧」
- 題目：
  `智慧工廠和傳統工廠最大的差別是什麼？`
- 選項：
  - A：工廠裡的機器顏色比較新
  - B：感測、控制、資料與設備可以一起協作判斷
  - C：工廠一定完全沒有人
  - D：只要有機械手臂就叫智慧工廠
- 正解：B
- Trait 分布：
  - B -> `automation`
  - A -> `maker`
  - C -> `operations`
  - D -> `control`

### Q2

- Type: `branching`
- 題目主軸：任務入口
- 題目：
  `警示燈亮起後，你想先從哪裡開始檢查？`
- 選項：
  - A：先看感測器和警示訊號
  - B：先看產線流程和機台動作
- 分支：
  - A -> `factory-q3-sensor`
  - B -> `factory-q3-line`
- Trait 分布：
  - A -> `inspection`
  - B -> `operations`

## 共通主線

### Q5

- Type: `matching`
- 題目主軸：設備與功能配對
- 題目：
  `請把工廠設備和最接近的功能配對起來。`
- 建議配對：
  - 感測器 -> 蒐集狀態資料
  - 檢測相機 -> 找出瑕疵或異常
  - 控制器 -> 協調機台反應
  - 輸送帶 / 機械手臂 -> 執行搬運或動作
- Trait 主軸：`automation`

### Q6

- Type: `ordering`
- 題目主軸：異常處理流程
- 題目：
  `如果產線出現異常，哪個處理順序最合理？`
- 建議排序：
  1. 先偵測異常
  2. 判斷是哪個環節出問題
  3. 調整設備或流程
  4. 再次檢查結果
  5. 確認正常後恢復運作
- Trait 主軸：`operations`

### Q7

- Type: `single-choice`
- 題目主軸：核心收束
- 題目：
  `為什麼智慧工廠也是電子工程師的重要舞台？`
- 選項：
  - A：因為只要有機器就一定和電子工程有關
  - B：因為感測、控制、檢測與自動化整合都需要電子工程能力
  - C：因為工廠比實驗室更熱鬧
  - D：因為工廠裡每台設備都一定是 AI
- 正解：B
- Trait 分布：
  - B -> `automation`
  - A -> `control`
  - C -> `maker`
  - D -> `inspection`

## Trait 設計

### Trait 1

- ID: `control`
- 代表方向：
  喜歡機台狀態、控制邏輯、穩定運作

### Trait 2

- ID: `inspection`
- 代表方向：
  喜歡細節觀察、異常辨識、品質判斷

### Trait 3

- ID: `automation`
- 代表方向：
  喜歡看見設備、資料與流程如何被整合成一套系統

### Trait 4

- ID: `operations`
- 代表方向：
  喜歡流程判斷、現場應變、維持產線節奏

## 結果卡草案

### 1. 產線控制指揮員

- 對應 Trait：`control`
- 副標籤建議：
  - `穩定判斷型`
  - `節奏掌控型`
- 文案方向：
  你擅長看懂機台與流程的節奏，會優先思考怎麼讓設備穩定、正確、持續地運作。

### 2. 品質檢測觀察員

- 對應 Trait：`inspection`
- 副標籤建議：
  - `細節敏感型`
  - `異常追蹤型`
- 文案方向：
  你很容易注意到細節差異與異常訊號，適合站在品質與檢測的第一線。

### 3. 自動化整合工程師

- 對應 Trait：`automation`
- 副標籤建議：
  - `系統串接型`
  - `協同配置型`
- 文案方向：
  你會自然去想不同設備、資料與控制邏輯怎麼串在一起，最像整合整條智慧產線的人。

### 4. 智慧維運調度員

- 對應 Trait：`operations`
- 副標籤建議：
  - `現場應變型`
  - `流程推進型`
- 文案方向：
  你會從任務進行和現場狀況出發，優先思考怎麼讓整個流程順下去。
