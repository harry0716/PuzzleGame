# Fourth Scene Draft v0.1

## 場景定位

- Scene ID: `smart-care-mission-station`
- 場景名稱：`智慧照護任務站`
- 主題：`智慧醫療 + 長照照護`
- 風格：更細緻的情境任務型場景，重視人物、照護關係與在地社會需求
- 玩家身份：智慧照護任務員

## 核心概念

第四場景不只是展示智慧醫療設備，而是要讓玩家理解：

- 科技如何幫助長者更安全
- 科技如何減輕照護者壓力
- 科技如何讓家屬更安心
- 科技如何協助長照與醫療體系更即時地發現問題

主軸不是「設備有多厲害」，而是：

**科技如何讓照護更有溫度、更有效率，也更有尊嚴。**

## 為什麼這個場景重要

這個場景很符合台灣當前社會情境，因為它能自然帶入：

- 高齡化
- 長照需求
- 跌倒風險
- 生理感測
- 遠距關懷
- 用藥提醒
- 智慧病房 / 智慧照護空間

它也會讓整個專案的場景組合更完整：

1. `晶片獵人`
   AI / 晶片 / 基礎系統認識
2. `電子工程雙體驗`
   展攤互動 / VR / 無人機 / 訊號整合
3. `智慧工廠任務站`
   自動化 / 感測 / 控制 / 產線邏輯
4. `智慧照護任務站`
   醫療照護 / 長照支持 / 以人為核心的科技應用

## 整體設計方向

這一個場景要比前三個更細緻，明確調整如下：

- 題數增加到 `10 題`
- 題目敘述更長
- 每題盡量帶人物和情境
- 更強調照護現場的判斷，而不是冷知識問答
- 文案更溫和，不做成「醫療考題」

## 場景故事

玩家被設定為一位智慧照護任務員，在一個結合長照與醫療照護的智慧照護中心中，協助觀察多位長者的狀況。

場景中的目標不是替代照護者，而是學會理解：

- 哪些警示真正重要
- 哪些感測資訊最值得先看
- 哪些裝置可以在不打擾長者的情況下提供幫助
- 當異常發生時，怎樣的流程最有助於照護安全

## 流程總覽

1. 開場：智慧照護真正想幫助的是什麼
2. 分支：先看生命徵象，或先看環境警示
3. 限時：快速判斷哪個提醒最需要先處理
4. 圖片：辨識智慧照護設備
5. 情境：跌倒 / 離床 / 長者安全
6. 配對：裝置與功能
7. 分支：家屬安心 or 照護效率，先處理哪種情況
8. 排序：照護異常事件處理流程
9. 情境：科技如何協助，但不能取代陪伴
10. 收束：智慧醫療與長照為何也是電子工程的重要舞台

## 題目草案方向

### Q1

- Type: `single-choice`
- 主軸：智慧照護最核心的目標
- 題目風格：
  用一位長者剛入住智慧照護中心的情境切入，讓玩家理解科技的目的不是監控，而是幫助安全與照護判斷。

### Q2

- Type: `branching`
- 主軸：任務入口
- 分支 A：
  先看生命徵象與感測資料
- 分支 B：
  先看環境與照護警示

### Q3

- Type: `timed-choice`
- 主軸：快速判斷哪一種提醒最值得先注意
- 方向：
  例如某位長者血氧異常、離床提醒、走動軌跡異常等

### Q4

- Type: `image-choice`
- 主軸：辨識智慧照護裝置
- 圖像方向可包含：
  - 智慧手環
  - 床邊感測器
  - 跌倒偵測設備
  - 一般非照護設備

### Q5

- Type: `single-choice`
- 主軸：跌倒偵測 / 離床提醒的照護價值
- 方向：
  強調長者安全與照護者即時反應

### Q6

- Type: `matching`
- 主軸：裝置功能配對
- 建議配對：
  - 血氧感測 -> 監測生理狀況
  - 離床提醒 -> 協助注意安全
  - 用藥提醒 -> 協助規律照護
  - 遠距照護平台 -> 讓家屬或照護團隊掌握狀況

### Q7

- Type: `branching`
- 主軸：照護現場優先順序
- 方向：
  例如同時出現家屬查詢、長者提醒、系統警示時，你先處理什麼

### Q8

- Type: `ordering`
- 主軸：異常事件處理流程
- 方向：
  發現警示 -> 確認狀況 -> 通知照護者 / 判讀資料 -> 協助處理 -> 回報與追蹤

### Q9

- Type: `single-choice`
- 主軸：科技和陪伴的界線
- 方向：
  強調智慧照護是協助，而不是取代人的陪伴與關懷

### Q10

- Type: `single-choice`
- 主軸：電子工程在智慧醫療與長照中的角色
- 方向：
  收束到感測、傳輸、警示、資料整合、設備設計等電子工程核心

## 題目敘事原則

這個場景的每題應盡量採用：

- 短情境
- 有人物
- 有照護目的
- 再提出問題

建議格式：

1. 描述一位長者或照護現場情境
2. 說明希望達成的照護目標
3. 再讓玩家選擇最合理的判斷或設備

## 建議 Trait

### Trait 1

- ID: `vitals`
- 代表方向：
  對生命徵象、感測資訊與身體狀況特別敏感

### Trait 2

- ID: `care`
- 代表方向：
  關注長者安全、陪伴感與照護品質

### Trait 3

- ID: `systems`
- 代表方向：
  擅長理解裝置、訊號、平台與資料如何整合

### Trait 4

- ID: `support`
- 代表方向：
  重視照護流程、團隊協作與實際協助

## 結果卡方向

### 1. 生命訊號觀察員

- 對應 Trait：`vitals`
- 副標籤建議：
  - `細膩判讀型`
  - `狀態敏感型`
- 文案方向：
  你很容易注意到生命徵象與身體狀況的小變化，適合站在第一線觀察風險與異常。

### 2. 智慧照護協調員

- 對應 Trait：`care`
- 副標籤建議：
  - `關懷整合型`
  - `安心守護型`
- 文案方向：
  你重視的不只是系統有沒有發出提醒，而是長者是否真的被照顧到、感受到安全與安心。

### 3. 醫療系統整合師

- 對應 Trait：`systems`
- 副標籤建議：
  - `訊號串接型`
  - `平台整合型`
- 文案方向：
  你最容易看見裝置、平台、警示與資料之間的關係，很像把整套照護系統串起來的人。

### 4. 長照科技守護員

- 對應 Trait：`support`
- 副標籤建議：
  - `現場支援型`
  - `流程陪伴型`
- 文案方向：
  你很在意照護現場是否順利、長者是否安心、家屬是否能放心，最能把科技轉化成真正的照護支持。

## 視覺方向建議

- 比前三個場景更柔和

  - 米白
  - 淺橘作為關懷提示色

圖像元素建議：

- 智慧手環
- 床邊感測器
- 長者房間 / 智慧病房
- 照護面板
- 家屬遠距關懷畫面
